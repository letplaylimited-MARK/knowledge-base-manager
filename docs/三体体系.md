# 三体体系.md

## 通用知识库框架三体集成指南

> 版本: V1.0 | 更新: 2026-04-21

### 一、三体系概述

通用知识库框架由三个核心体系协同工作构成完整的知识管理闭环：

1. **知识图谱体系** - 结构化知识的表达和组织
2. **记忆系统** - 长期保持和检索知识的能力  
3. **AI协作系统** - 智能体之间的协作和任务执行

这三个体系不是孤立的，而是通过well-defined接口紧密集成，形成知识的获取→存储→应用的完整循环。

### 三、每个体系的职责和接口

#### 1. 知识图谱体系（05-知识沉淀/wiki/）
**职责**：
- 概念定义和关系建模
- 实体属性和关联管理
- 知识的结构化表示和推理
- 本体论维护和演化

**接口**：
- **输入**：来自知识结晶层的结构化知识（概念、关系、属性）
- **输出**：提供语义查询能力、知识推理服务
- **存储格式**：Markdown文件+结构化元数据
- **查询方式**：基于图遍历和属性匹配

#### 2. 记忆系统（.workbuddy/记忆层/）
**职责**：
- 长期保持知识和经验
- 上下文关联和情境检索
- 知识遗忘曲线管理
- 个人/团队知识积累

**接口**：
- **输入**：来自知识图谱的验证知识、来自AI协作的执行结果
- **输出**：基于相关性和时间衰减的知识检索
- **核心引擎**：memoryos.py（基于HNSW的向量存储+知识图谱）
- **存储层**：短期/中期/长期记忆分层

#### 3. AI协作系统（.workbuddy/AI协作体系/）
**职责**：
- 智能体角色定义和能力注册
- 任务分解和智能体调度
- 多智能体协作流程管理
- 结果整合和质量控制

**接口**：
- **输入**：用户任务请求、来自记忆系统的相关知识
- **输出**：执行计划、中间结果、最终交付物
- **核心机制**：技能(skill)系统和智能体池
- **协作模式**：顺序、并行、层级分解

### 四、关键集成点

#### 1. 知识摄取流程（Input → KG）
```
外部文档 → 01-收件箱 → 知识结晶 → 
    ↓
知识图谱（概念实体抽升） → 记忆系统（经验沉淀）
```

#### 2. 知识检索流程（Query → Output）
```
用户查询 → 记忆系统（上下文相关记忆） → 
    ↓
知识图谱（语义相似概念） → AI协作系统（任务分解执行） → 
    ↓
生成回答 → 记忆系统（强化相关知识） → 输出
```

#### 3. 知识更新流程（Learning → Storage）
```
执行反馈 → AI协作系统（结果评估） → 
    ↓
知识图谱（新关系/属性） → 记忆系统（重要经验） → 
    ↓
更新索引 → 下次查询受益
```

### 五、一致性保证机制

为了确保三体系数据的一致性，框架提供了以下机制：

1. **事务性更新**：重要操作会同步更新所有相关体系
2. **版本控制**：每个体系内部通过Git进行版本追踪
3. **一致性检查**：定期运行一致性验证脚本
4. **故障恢复**：通过日志和checkpoint机制实现快速恢复

#### 示例一致性操作：
```python
# 当添加新概念时，需要同步更新：
1. 知识图谱：05-知识沉淀/wiki/concepts/新概念.md
2. 记忆系统：.workbuddy/记忆层/经验库/新概念经验.md  
3. AI知识库：.workbuddy/AI协作体系/模型配置/new-concept.yaml
4. 索引更新：运行 .workbuddy/scripts/update_index.py
```

### 六、使用指南

#### 对于知识维护者：
1. 新知识首先进入01-收件箱
2. 使用思绪整理系统.md进行初步结构化
3. 将结晶知识提交到对应的wiki/目录
4. 更新索引：python .workbuddy/scripts/update_index.py
5. 更新AI能力库（如果需要新技能）

#### 对于普通用户：
1. 直接提出问题或提供输入
2. 系统自动通过三体系协同处理
3. 回答将融合知识图谱事实、个人经验和AI推理
4. 重要互动会自动沉淀到记忆系统

#### 对于系统管理者：
1. 定期更新索引：python .workbuddy/scripts/update_index.py
2. 监控三体系性能指标
3. 根据使用模式调整存储策略
4. 维护和升级核心组件（memoryos.py等）

### 七、演示场景

#### 场景1：新概念学习
```
用户：什么是Transformer架构？
→ 知识图谱：检索到Transformer概念定义
→ 记忆系统：找到用户之前的深度学习学习记录
→ AI协作：调用技能生成解释性回答
→ 输出：结合定义、个人学习历史和AI生成的类比解释
→ 反馈：用户确认有效，系统强化Transformer在记忆中的关联度
```

#### 场景2：经验复用
```
用户：如何优化我的SQL查询？
→ 记忆系统：找到用户过去成功的SQL优化案例
→ 知识图谱：检索到SQL优化的通用原则和模式
→ AI协作：结合两者生成针对性的优化建议
→ 输出：个人经验+通用方法+AI定制方案
→ 沉淀：新的优化经验被加入到用户的SQL优化记忆中
```
