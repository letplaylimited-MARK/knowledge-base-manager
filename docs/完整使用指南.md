## 一、模型概览

### 系统架构

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  01-收件箱/          │  知识摄入层                        │
│  (inbox_watcher.py)  │  文件监控 + 自动分类              │
├─────────────────────────────────────────────────────────┤
│  03-资产库/          │  加工处理层                        │
│  04-输出成果/        │  内容分析 + 命名优化 + 文件归档   │
├─────────────────────────────────────────────────────────┤
│  05-知识沉淀/wiki/   │  知识结晶层                        │
│  (concepts/entities  │  概念/实体/来源/比较 四维知识图谱 │
│   sources/comparisons)                                   │
├─────────────────────────────────────────────────────────┤
│  .workbuddy/         │  持久化层                         │
│   ├─ 记忆层/         │  记忆引擎 (短/中/长期)            │
│   ├─ AI协作体系/     │  多模型智能体协作                 │
│   └─ 流程/           │  工作流标准 + 验证体系             │
├─────────────────────────────────────────────────────────┤
│  AI检索 + 命令行      │  输出层                          │
│  (search_content.py) │  语义搜索 + 知识图谱查询          │
└─────────────────────────────────────────────────────────┘
```

### 核心子系统

| 子系统 | 位置 | 职责 |
|--------|------|------|
| 知识图谱 | `05-知识沉淀/wiki/` | 概念/实体/来源/比较的结构化知识 |
| 记忆系统 | `.workbuddy/记忆层/` | 跨会话短期→中期→长期记忆 |
| AI协作 | `.workbuddy/AI协作体系/` | 多模型调度、角色协作、智能体池 |
| 工作流 | `.workbuddy/流程/` | 任务验证、交接规范、标准流程 |

## 二、安装指南

### 2.1 系统要求

| 项目 | 最低要求 | 推荐 |
|------|----------|------|
| Python | 3.8+ | 3.11+ |
| 磁盘空间 | 100 MB | 1 GB+（含知识库） |

| Git | 可选（版本管理） | 推荐安装 |

### 2.2 安装步骤

**Windows:**
```bash
# 1. 进入框架目录
cd D:\AI-聊天工作专项整理\07-项目专区\数据库管理

# 2. 一键安装（推荐）
00-快速开始\setup.bat

# 3. 或手动安装
pip install -r requirements.txt
python .workbuddy\scripts\update_index.py
```

**macOS / Linux:**
```bash
# 1. 进入框架目录
cd /path/to/database-management

# 2. 安装依赖
pip3 install -r requirements.txt

# 3. 构建知识索引
python3 .workbuddy/scripts/update_index.py

# 4. 创建必要目录
mkdir -p "01-收件箱/待处理"
mkdir -p "02-对话记录"
mkdir -p "04-输出成果/报告"
```

### 2.3 验证安装

## 四、使用指南

### 4.1 知识录入

#### 方式A：通过收件箱（推荐）

```bash
# 1. 将文件放入 01-收件箱/
#    支持格式: .md .txt .docx .xlsx .csv .json .html .pdf

# 2. 运行收件箱扫描
python .workbuddy/scripts/inbox_watcher.py

# 3. 查看入库建议
#    生成报告在: 01-收件箱/待处理/*-入库建议.md

# 4. 确认后移动文件到目标目录
#    或让 auto_organizer.py 自动处理
python .workbuddy/scripts/auto_organizer.py
```

#### 方式B：手动录入wiki

```bash
# 概念定义 → concepts/
echo "# 概念名" > "05-知识沉淀/wiki/concepts/概念名.md"

# 实体定义 → entities/
echo "# 实体名" > "05-知识沉淀/wiki/entities/实体名.md"

# 知识来源 → sources/
echo "# 来源标题" > "05-知识沉淀/wiki/sources/来源标题.md"

# 比较分析 → comparisons/
echo "# 比较主题" > "05-知识沉淀/wiki/comparisons/比较主题.md"
```

#### 方式C：批量流水线处理

```bash
# 适合大量文档导入
python .workbuddy/scripts/file_processing_pipeline.py
# 自动完成：内容理解 → 智能摘要 → 版本管理 → 命名决策
```

## 六、注意事项

### 6.1 路径与执行

| 注意点 | 说明 |
|--------|------|
| **工作目录** | 所有脚本需在框架根目录执行 (`数据库管理/`) |
| **路径自动解析** | 路径基于 `Path(__file__).resolve().parent.parent.parent` 动态计算，**不存在硬编码** |
| **Windows路径** | 使用 `\` 或 `/` 均可，Python Path 对象自动处理 |
| **Python版本** | 11个脚本已统一路径策略，无需手动修改 WORKSPACE |

### 6.2 脚本执行顺序依赖

```
inbox_watcher.py  ← 可独立运行
enhanced_inbox_watcher.py  ← 可独立运行
     │
     ▼
file_scanner.py ← inbox_watcher 产出为输入
relation_discovery.py ← file_scanner 产出为输入
     │
     ▼
auto_organizer.py ← 依赖收件箱有文件
     │
     ▼
content_analyzer.py / naming_optimizer.py ← 依赖文件已被分类
     │
     ▼
update_index.py ← 依赖知识目录有内容
     │
     ▼
search_content.py ← 依赖索引已构建
```

### 6.3 数据一致性原则

- **知识添加 → 索引更新**: 每次新增/修改知识后，运行 `update_index.py`
- **文件移动 → 关联监控**: 移动文件后，运行 `project_relationship_manager.py` 更新关系
- **修改系统配置 → 测试验证**: 修改 `.workbuddy/` 下配置后，运行关联脚本验证
- **定期维护**: 至少每周执行一次 `maintenance_task.py`

### 6.4 安全性

| 风险 | 防范措施 |
|------|----------|
| API密钥泄露 | 使用 `${环境变量}` 引用密钥，**禁止**硬编码在 YAML 中 |
| 文件丢失 | 自动备份保留最近5份，建议配合 Git 版本管理 |
| 误删文件 | `99-归档/` 目录作缓冲，不直接删除 |
| 敏感信息 | 日志目录排除 `.workbuddy/` 外内容扫描 |


```bash
# 再搜索

```

```bash

dir 01-收件箱
# 检查待处理目录是否存在
dir 01-收件箱\待处理
```

**Q: 如何在 Mac/Linux 上运行？**
```bash
# 脚本使用纯 Python，跨平台兼容
# 路径分隔符 Python Path 对象自动适配
# setup.bat 仅限 Windows，其他系统手动执行 pip install + mkdir
```

**Q: 如何添加新的知识类型？**
```bash
# 1. 在 wiki/ 下创建新目录
```
