# 结构体系.md

## 通用知识库框架结构定义

> 版本: V2.0 | 更新: 2026-05-23

### 一、目录结构总览

```
D:/数据库管理/
├── 00-快速开始/           # 新用户入门引导
├── 01-收件箱/             # 新知识入口（待处理）
├── 02-对话记录/           # 对话存档与会议纪要
├── 03-资产库/             # 可复用资产（Prompt、模板等）
├── 04-输出成果/           # 产出文件（报告、文档等）
├── 05-知识沉淀/           # 知识库核心（结构化知识）
│   └── wiki/              # 知识图谱
│       ├── concepts/      # 概念定义
│       ├── entities/      # 实体定义
│       ├── sources/       # 源摘要
│       └── comparisons/   # 比较分析
├── 06-参考资料/           # 外部参考材料
├── 07-项目文档/           # 项目特定文档
├── 99-归档/               # 历史版本备份
└── .workbuddy/            # 系统配置核心
    ├── 记忆层/            # 长期记忆系统
    ├── AI协作体系/        # 智能体角色定义
    ├── 流程/              # 工作流标准
    └── config/            # 系统配置
```

### 二、核心子系统划分

#### 1. 知识图谱系统（05-知识沉淀/）
- **概念层**：抽象概念定义和关系
- **实体层**：具体业务实体和属性
- **来源层**：知识溯源和引用追踪
- **比较层**：多维度对比分析

#### 2. 记忆系统（.workbuddy/记忆层/）
- **短期记忆**：当前会话上下文
- **中期记忆**：进行中的项目知识
- **长期记忆**：持久化的经验和共识

#### 3. AI协作系统（.workbuddy/AI协作体系/）
- **角色定义**：不同AI智能体的职责
- **协作流程**：多智能体任务分解
- **能力注册**：工具和技能的统一管理

#### 4. 工作流系统（.workbuddy/流程/）
- **任务验证**：质量检查标准
- **交接机制**：知识无缝传递
- **思绪整理**：碎片化想法的结构化
- **智能分流**：基于内容的自动路由

### 三、数据流向说明

```
用户输入 → 01-收件箱 → 知识结晶 → 05-知识沉淀/wiki
                              ↘
                               → .workbuddy/记忆层 → 长期记忆
                              ↘
                               → .workbuddy/AI协作体系 → 智能处理
                              ↘
                               → 04-输出成果 → 可视化报告
```

### 四、扩展点和自定义方式

1. **知识图谱扩展**：在wiki/下添加新的概念类型
2. **记忆系统**：通过memoryos.py配置不同的存储策略
3. **AI角色**：在AI协作体系/中定义新的智能体类型
4. **工作流**：在流程/目录添加新的处理管线
