# 验证指南

> 版本：V2.0 | 更新：2026-05-23

本指南说明如何使用通用知识库框架的验证体系，确保任务执行的质量和完整性。

## 🎯 验证体系概览

框架提供了完整的任务验证流程，包含四个维度的检查：
1. **功能验证** - 确认需求是否被正确理解和完整实现
2. **质量验证** - 检查内容准确性、逻辑完整性和表达质量
3. **交接验证** - 验证产出物归档、文档生成、代码提交和记忆更新
4. **风险验证** - 识别潜在风险、说明依赖项和记录已知限制

## 📋 使用验证体系的时机

验证应在以下情况下执行：
- 任务执行完成后
- 重要文件创建或修改后
- 关键决策做出后
- 每日工作结束时
- 用户主动请求验证时
- 自己感觉需要检查时

## 🔧 验证操作步骤

### 步骤1：功能验证（必须）
检查任务是否正确理解并完整实现了用户需求：
```
[ ] 需求理解正确（复述用户需求）
[ ] 功能完整实现（对比需求清单）
[ ] 输出格式正确（符合约定）
[ ] 无明显错误（人工检查）
```

### 步骤2：质量验证（必须）
验证输出内容的质量和专业性：
```
[ ] 内容准确无误（事实核查）
[ ] 逻辑完整清晰（结构检查）
[ ] 语言表达通顺（阅读测试）
[ ] 无冗余遗漏（完整性）
```

### 步骤3：交接验证（必须）
确保任务结果能够正确交接和保存：
```
[ ] 产出物已归档（指定目录）
[ ] 交接文档已生成（模板填充完毕）
[ ] Git已提交（带清晰说明）
[ ] 记忆已更新（相关文件已同步到记忆系统）
```

### 步骤4：风险验证（必须）
识别和记录潜在问题和后续行动：
```
[ ] 潜在风险已标注
[ ] 依赖项已说明
[ ] 已知限制已记录
[ ] 后续行动已明确
```

## ⚙️ 自动化验证工具

框架提供了一些自动化工具辅助验证过程：

### 索引更新验证
运行以下命令检查知识索引是否需要更新：
```bash
python .workbuddy\scripts\update_index.py
```

### 索引完整性检查
检查知识库索引是否完整：
```bash
python .workbuddy\scripts\update_index.py --check
```

### 内容搜索验证
验证搜索功能是否正常：
```bash
python .workbuddy\scripts\search_content.py "验证关键词"
```

## 🏆 最佳实践

1. **即时验证**：任务完成后立即进行验证，不要拖延
2. **逐项检查**：按照功能→质量→交接→风险的顺序进行验证
3. **记录问题**：发现任何问题都要详细记录在验证报告中
4. **用户参与**：尽可能让最终用户参与验证的确认步骤
5. **持续改进**：将验证中发现的问题作为系统优化的输入
6. **知识沉淀**：验证通过的经验和最佳实践要沉淀到记忆系统

## ❓ 常见问题

**Q: 验证需要花费多长时间？**
A: 简单任务验证通常需要5-15分钟，复杂任务可能需要更长时间。时间投入能显著提高交付质量。

**Q: 如果验证失败怎么办？**
A: 首先完成当前步骤的修复工作，然后从失败的步骤重新开始验证流程。

**Q: 需要对团队成员的工作进行验证吗？**
A: 是的，建议在团队协作中使用相同的验证标准，确保输出质量一致。

**Q: 如何处理验证中的主观判断？**
A: 尽可能使用具体可衡量的标准，对于主观事项，记录判断依据并寻求第二方确认。

**Q: 验证报告应该保存在哪里？**
A: 验证报告应与任务相关联，可以保存在任务文档中或02-对话记录/目录下。
