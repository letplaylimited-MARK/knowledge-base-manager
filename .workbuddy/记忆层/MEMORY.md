## 三、记忆类型定义

### 3.1 情景记忆 (Episodic)

具体事件和经历

**存储位置**: `05-知识沉淀/wiki/sources/`

**格式**:
```markdown
# 事件标题

> 时间：YYYY-MM-DD
> 参与者：[角色]
> 场景：[场景描述]

## 事件记录
[详细记录]

## 学到的经验
[经验总结]
```

### 3.2 语义记忆 (Semantic)

概念、定义和知识

**存储位置**: `05-知识沉淀/wiki/concepts/` + `05-知识沉淀/知识结晶/`

**格式**:
```markdown
# 概念名称

## 定义
[清晰的定义]

## 关联概念
- [[concepts/相关概念1]]
- [[entities/相关实体]]

## 示例
[使用示例]
```

### 3.3 程序记忆 (Procedural)

工作流程和最佳实践

**存储位置**: `.workbuddy/流程/`

**格式**:
```markdown
# 流程名称

## 步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 适用场景
- [场景1]
- [场景2]

## 注意事项
[关键点]
```

### 3.4 情感记忆 (Emotional)

用户偏好和交互风格

**存储位置**: `.workbuddy/记忆层/user_profile.md`

**格式**:
```markdown
# 用户画像

## 偏好
- [偏好1]
- [偏好2]

## 交互风格
- [风格描述]

## 历史交互
| 时间 | 任务 | 结果 |
|------|------|------|
```

## 四、更新规则

### 4.1 即时更新

发生以下情况时**立即更新**：
- 完成重要任务 → 写入任务记忆
- 产生新知识结晶 → 写入知识沉淀
- 用户明确偏好 → 写入用户画像
- 系统决策 → 写入决策日志

### 4.2 定期更新

| 周期 | 内容 | 位置 |
|------|------|------|
| 每日 | 工作日志 | `wiki/log.md` |
| 每周 | 知识总结 | `知识结晶/周总结/` |
| 每月 | 系统回顾 | `.workbuddy/迭代日志/` |

### 4.3 触发更新

| 操作 | 触发更新 |
|------|----------|
| 完成文件编辑 | → 任务记忆 |
| 完成知识结晶 | → 知识图谱 |
| 完成项目交付 | → 经验库 |
| 用户提供偏好 | → 用户画像 |
| 系统做决策 | → 决策日志 |

## 六、与其他系统集成

### 6.1 与AGENTS.md集成

```
AGENTS.md
    ↓
    定义AI行为规范
    ↓
    当需要上下文时
    ↓
    读取 MEMORY.md ← 获取长期记忆
    ↓
    返回相关上下文
```

### 6.2 与工作流集成

```
工作流执行
    ↓
    各阶段触发记忆更新
    ↓
    任务记忆 → 经验库 → 知识图谱
    ↓
    形成完整的记忆闭环
```
