# 任务交接规范

> 经验库 · 核心规范 · 2026-04-11

---

## 目的

确保任务在AI之间、人机之间可靠交接，实现连续执行。

---

```markdown

> 状态：进行中/已完成/已归档

[一句话描述这是什么任务]

- 当前阶段：
- 完成度：%

---

## 交接检查清单

- [ ] 任务概述清晰
- [ ] 当前进度明确
- [ ] 任务清单完整
- [ ] 待完成工作已列出
- [ ] 决策原因已记录
- [ ] 风险已评估
- [ ] 下一步行动明确
- [ ] 交付物清单完整

满足全部 → 可以交接 ✓
不满足 → 补充后再交接
```

---

## 使用场景

### 场景1：AI之间交接

```
AI-A完成任务的一部分 → 生成交接文档 → AI-B读取 → 继续执行
```

### 场景2：人机交接

```
AI执行 → 产出交接文档 → 用户确认 → 继续/修改/终止
```

### 场景3：会话之间交接

```
本次会话结束 → 保存交接文档 → 下次会话读取 → 继续工作
```

| 状态 | 归档位置 |

| 已完成 | `任务记忆/已完成/` |
