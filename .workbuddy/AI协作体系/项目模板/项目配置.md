# 项目模板 - 通用配置

> **版本**: V1.0
> **适用**: 所有新项目

---

## 项目基本信息

```yaml
project:
  name: "[项目名称]"
  type: "运营/开发/研究/整理"
  status: "启动中"
  start_date: "2026-04-11"
  
  description: |
    项目简要描述
    
  goals:
    - 目标1
    - 目标2
    
  constraints:
    - 约束1
    - 约束2
```

## 目录结构

```
[项目名]/
├── 01-收件箱/              ← 新文件入口
├── 02-对话记录/            ← AI对话存档
├── 03-资料库/              ← 业务资料
│   ├── 制度文档/
│   ├── 操作手册/
│   └── 参考资料/
├── 04-产出物/              ← 产出成果
│   ├── 报告/
│   ├── 方案/
│   └── 代码/
├── 05-归档/                ← 历史归档
├── .workbuddy/             ← 系统配置
│   ├── 项目配置.md
│   ├── 技能配置.md
│   ├── 知识结构.md
│   └── 迭代日志/
└── README.md               ← 项目说明
```

## 技能配置

```yaml
skills:
  # 通用技能（必选）
  required:
    - 全文检索
    - 文件整理
    - 报告生成
    - 知识管理
    
  # 专业技能（按需）
  optional:
    - "[专业技能1]"
    - "[专业技能2]"
    
  # 实验技能（可选）
  experimental:
    - "[实验技能]"
```

---

## 智能体配置

```yaml
agents:
  # 核心智能体
  core:
    - 协调员（始终启用）
    
  # 工作智能体（按需）
  workers:
    - 扫描员（信息收集）
    - 分析员（深度分析）
    - 推荐员（方案推荐）
```

---

## 知识结构

```yaml
knowledge:
  # 项目专属知识
  project_specific:
    - 术语表
    - 概念定义
    - 规则体系
    
  # 关联知识
  related:
    - 关联项目1
    - 关联项目2
    
  # 共享知识
  shared:
    - 通用方法论
    - 最佳实践
```

## 工作流程

```yaml
workflow:
  main: "飞轮工作流"
  steps:
    - Scan      # 扫描
    - Diagnose  # 诊断
    - Plan      # 规划
    - Execute   # 执行
    - Validate  # 验证
    - Crystallize # 沉淀
    
  shortcuts:
    - "快速整理" → 批量处理
    - "快速查询" → 检索
    - "快速报告" → 生成报告
```

# （手动复制本文件夹内容）
