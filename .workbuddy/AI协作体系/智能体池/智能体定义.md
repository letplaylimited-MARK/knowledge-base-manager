## 智能体定义结构

每个智能体应包含以下配置部分：

```yaml
# 智能体基本信息
agent_name: "智能体名称"
agent_type: "智能体类型"  # coordinator, scanner, analyzer, recommender, custom
description: "智能体功能描述"

# AI模型配置
model_config:
  model_name: "使用的AI模型名称"  # 对应模型配置目录中的文件名（不含.yaml后缀）
  provider: "模型提供商"  # 可选，用于验证
  # 例如: model_name: "openai-gpt4"  # 将使用 模型配置/openai-gpt4.yaml

# 能力定义
capabilities:
  - "能力1"
  - "能力2"
  - "能力3"

# 触发条件
trigger: "触发此智能体的条件描述"
skill_level: "所需技能水平"  # 初级, 中级, 高级, 专家
```

## 1. 协调员（Coordinator）

### 角色定义
**总指挥** - 所有任务的入口和出口

### 核心能力
```yaml
agent_name: "协调员"
agent_type: "coordinator"
description: "负责任务分配、进度协调和结果整合的智能体"

model_config:
  model_name: "openai-gpt4"  # 使用GPT-4进行复杂决策
  provider: "openai"

capabilities:
  - 理解用户需求
  - 制定执行计划
  - 协调多智能体协作
  - 整合多智能体结果
  - 沟通协调

trigger: "有新任务需要处理时"
skill_level: "专家"
```

### 工作流程
```
协调员执行流程：
    1. 接收用户需求
    2. 分析任务复杂度和所需能力
    3. 制定详细的执行计划
    4. 根据能力匹配分配任务给合适的智能体
    5. 监控执行进度
    6. 整合所有智能体的输出结果
    7. 向用户反馈最终结果
```

### 使用场景
- 复杂任务规划
- 多智能体协作场景
- 需要高级决策的情况
- 跨领域知识整合

## 2. 扫描员（Scanner）

### 角色定义
**信息收集专家** - 快速获取和整理信息

### 核心能力
```yaml
agent_name: "信息扫描员"
agent_type: "scanner"
description: "负责信息收集、文档解析和数据提取的智能体"

model_config:
  model_name: "anthropic-claude3"  # 使用Claude进行精准信息理解
  provider: "anthropic"

capabilities:
  - 文档内容扫描
  - 信息提取和分类
  - 格式转换和标准化
  - 噪声过滤和去重
  - 元数据生成

trigger: "有新信息需要处理时"
skill_level: "中级"
```

### 工作流程
```
信息扫描员执行流程：
    1. 接收原始信息或文档
    2. 预处理和格式标准化
    3. 扫描提取关键信息和元数据
    4. 信息分类和标签生成
    5. 去重和噪声过滤
    6. 生成结构化的信息摘要
    7. 将处理结果发送给分析智能体
```

### 适用场景
- 大规模文档处理
- 信息抽取和结构化
- 数据清洗和预处理
- 元数据生成和标签化

## 3. 分析员（Analyzer）

### 角色定义
**深度分析专家** - 发现规律和洞察

### 核心能力
```yaml
agent_name: "分析员"
agent_type: "analyzer"
description: "负责数据分析、模式识别和深度洞察的智能体"

model_config:
  model_name: "zhipu-glm4"  # 使用智谱GLM-4进行深度分析
  provider: "zhipu"

capabilities:
  - 数据趋势分析
  - 模式识别和异常检测
  - 关联关系发现
  - 预测和建模
  - 报告生成和可视化建议

trigger: "有数据需要分析时"
skill_level: "高级"
```

### 工作流程
```
分析员执行流程：
    1. 接收结构化或半结构化数据
    2. 数据预处理和清洗
    3. 应用统计方法和机器学习算法
    4. 识别模式、趋势和异常点
    5. 生成分析报告和可视化建议
    6. 提出改进建议和后续行动方案
```

### 适用场景
- 业务数据分析
- 市场趋势预测
- 异常检测和风险评估
- 科研数据分析和论文写作辅助

## 4. 推荐员（Recommender）

### 角色定义
**方案推荐专家** - 生成最优方案

### 核心能力
```yaml
agent_name: "推荐员"
agent_type: "recommender"
description: "负责生成个性化推荐和决策建议的智能体"

model_config:
  model_name: "yi-light"  # 使用零一万物Yi系列进行高效推荐
  provider: "yi"

capabilities:
  - 个性化内容推荐
  - 决策支持和方案选择
  - 风险评估和预警
  - 资源匹配和优化
  - 用户行为建模

trigger: "需要生成推荐或建议时"
skill_level: "中级"
```

### 工作流程
```
推荐员执行流程：
    1. 接收用户需求和上下文信息
    2. 分析用户历史行为和偏好
    3. 匹配可用资源和方案选项
    4. 应用推荐算法生成候选列表
    5. 根据多个维度进行评分和排序
    6. 生成个性化推荐理由和解释
    7. 提供后续行动建议和跟踪方案
```

### 适用场景
- 个性化新闻和内容推荐
- 产品和服务推荐
- 投资理财建议
- 学习路径规划和课程推荐
- 职业发展建议和技能学习路线

## 智能体链式协作模式

### 标准协作流程
```
[用户需求] 
    → [协调员] 了解需求并制定计划
    → [信息扫描员] 收集和处理相关信息
    → [分析员] 深度分析处理后的数据
    → [推荐员] 生成基于分析的建议和方案
    → [协调员] 整合结果并反馈给用户
```

### 反馈与改进机制
```yaml
feedback_mechanism:
  sources:
    - 用户最终评价
    - 中间过程质量检查
    - 系统性能监控
    
  metrics:
    - 任务完成时间
    - 结果准确率
    - 用户满意度
    - 资源利用效率
    
  improvement:
    - 模型参数调整
    - 工作流优化
    - 知识库更新
    - 智能体能力增强
```

## 模型配置使用说明

### 如何指定AI模型
在智能体定义中的 `model_config` 部分指定要使用的AI模型：

```yaml
model_config:
  model_name: "model-identifier"  # 对应 模型配置/ 目录下的 {model-identifier}.yaml
  provider: "model-provider"      # 可选字段用于验证
```

### 示例
- 使用OpenAI GPT-4: `model_name: "openai-gpt4"`
- 使用Anthropic Claude 3: `model_name: "anthropic-claude3"`  
- 使用智谱GLM-4: `model_name: "zhipu-glm4"`
- 使用零一万物Yi: `model_name: "yi-light"`
- 使用Ollama Llama 3: `model_name: "ollama-llama3"`

### 配置文件位置
所有模型配置文件存放在：
`.workbuddy\AI协作体系\模型配置\`
对应的文件格式为：`{model_name}.yaml`

### 环境变量支持
为保护API密钥等敏感信息，配置文件支持环境变量引用：
```yaml
api_key: "${OPENAI_API_KEY}"
```
系统将在加载配置时自动替换环境变量。
