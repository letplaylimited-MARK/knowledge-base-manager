# AI协调协作系统

> **版本**: V2.0  
> **更新时间**: 2026-05-17

## 系统概述

AI协调协作系统是通用知识库框架的核心组件，负责智能体的创建、配置、调度和协作。本系统支持多种AI大模型的灵活接入和智能体角色的标准化定义。

## 核心组件

### 1. 智能体池
存放各类预定义智能体模板和自定义智能体定义
- 路径: `.workbuddy/AI协作体系/智能体池/`
- 主要文件: `智能体定义.md` - 智能体配置模板和使用说明

### 2. 角色库
定义不同AI智能体的角色和职责
- 路径: `.workbuddy/AI协作体系/角色库/`
- 包含: 分析师.md, 推荐员.md, 架构师.md, 观察员.md 等角色定义

### 3. 项目模板
提供常见项目的智能体配置模板
- 路径: `.workbuddy/AI协作体系/项目模板/`
- 主要文件: `项目配置.md` - 项目级智能体配置示例

### 4. 模型配置系统 (新增)
支持灵活配置各种AI大模型的接入参数
- 路径: `.workbuddy/AI协作体系/模型配置/`
- 主要文件:
  - `模型配置说明.md` - 模型配置系统说明
  - `openai-gpt4.yaml` - OpenAI GPT-4配置示例
  - `anthropic-claude3.yaml` - Anthropic Claude 3配置示例

  - `yi-light.yaml` - 零一万物Yi-Light配置示例
  - 更多模型配置可根据需求添加

### 5. AI协作核心文档
- `工作流程.md` - 标准工作流程定义
- `角色定义.md` - 角色职责详细说明
- `知识图谱.md` - 知识图谱在AI协作中的应用
- `收件箱规则.md` - 收件箱处理规则和自动化
- `交接清单.md` - 任务交接标准和检查清单
- `MEMORY.md` - AI协作系统的记忆使用说明

## 系统特性

### 1. 模型无关设计
通过模型配置系统，智能体可以轻松切换不同的AI大模型：
- 支持OpenAI系列 (GPT-3.5, GPT-4等)
- 支持Anthropic系列 (Claude 2, Claude 3等)
- 支持Google系列 (Gemini Pro, Gemini Ultra等)
- 支持Meta系列 (Llama 2, Llama 3等)
- 支持深度求索系列 (DeepSeek-V2等)
- 支持智谱AI系列 (GLM-4等)
- 支持百川系列 (Baichuan2, Baichuan3等)
- 支持零一万物系列 (Yi系列等)
- 支持本地部署模型 (Ollama, LMStudio, vLLM等)
- 支持自定义API端点

### 2. 智能体角色标准化
预定义了四种核心智能体角色：
- **协调员 (Coordinator)**: 任务分配、进度协调、结果整合
- **信息扫描员 (Scanner)**: 信息收集、文档解析、数据提取
- **分析员 (Analyzer)**: 数据分析、模式识别、深度洞察
- **推荐员 (Recommender)**: 个性化推荐、决策支持、方案选择

### 3. 灵活的工作流支持
支持多种智能体协作模式：
- 链式协作: 协调员 → 信息扫描员 → 分析员 → 推荐员 → 协调员
- 并行处理: 多智能体同时处理不同任务
- 动态调度: 根据任务需求临时分配智能体
- 反馈循环: 基于执行结果持续优化智能体配置

### 4. 安全的配置管理
- 支持环境变量引用保护敏感信息 (API密钥等)
- 配置文件与代码分离，便于维护和更新
- 统一的配置格式降低学习成本
- 示例配置文件帮助用户快速上手

## 使用方法

### 1. 配置AI模型
在 `.workbuddy/AI协作体系/模型配置/` 目录中创建或编辑模型配置文件：
- 参考示例文件如 `openai-gpt4.yaml`
- 使用环境变量保护敏感信息: `api_key: "${OPENAI_API_KEY}"`
- 根据需求调整模型参数 (temperature, max_tokens等)

### 2. 定义或修改智能体
编辑 `.workbuddy/AI协作体系/智能体池/智能体定义.md` 或创建新的智能体定义：
- 在 `model_config` 部分指定要使用的模型名称
- 例如: `model_name: "openai-gpt4"` 将使用对应的YAML配置文件
- 定义智能体的能力、触发条件和执行流程

### 3. 使用智能体
通过系统接口调用已配置的智能体：
- 系统将根据智能体定义自动加载对应的AI模型配置
- 智能体将使用指定的模型进行推理和处理
- 结果将按照定义的执行流程返回

## 配置最佳实践

### 安全建议
1. 使用环境变量存储API密钥和其他敏感信息
2. 定期轮换API密钥
3. 限制API密钥的权限范围（只授予必要的访问权限）
4. 监控API使用情况以检测异常

### 性能优化
1. 根据任务类型选择合适的模型（复杂推理用强模型，简单任务用快模型）
2. 调整temperature参数平衡创造性和一致性
3. 设置合理的max_tokens避免过长输出
4. 使用流式输出改善交互体验（当模型支持时）

### 维护建议
1. 定期更新模型配置以适应新模型发布
2. 监控不同模型的性能和成本
3. 收集用户反馈持续优化智能体定义
4. 备份重要的配置文件

## 示例工作流

以下是一个典型的知识处理工作流示例：

1. **用户提出问题** → 系统触发协调员
2. **协调员使用openai-gpt4** 分析问题并制定计划
3. **信息扫描员使用anthropic-claude3** 从知识库中提取相关信息
4. **分析员使用zhipu-glm4** 对信息进行深度分析
5. **推荐员使用yi-light** 生成个性化建议和方案
6. **协调员使用openai-gpt4** 整合所有结果并反馈给用户

这种模型混合使用策略可以在保证质量的同时优化成本和响应速度。
