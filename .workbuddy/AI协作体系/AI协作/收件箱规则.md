# 收件箱规则 - 自动处理规范

> **这是AI处理新文件的"处理手册"**
> **定义了文件类型判断、分类标准、处理流程**

---

## 一、收件箱定位

### 1.1 收件箱的作用

```markdown
## 什么是收件箱

📥 用户放资料的地方
📋 AI处理文件的入口
🔄 资料变知识的起点

## 收件箱 ≠ 垃圾堆

❌ 不要什么都往里放
✅ 放需要整理的资料
✅ 放需要处理的素材
```

### 1.2 使用规则

```markdown
## 可以放收件箱

✅ 新收集的资料
✅ 需要整理的文件
✅ 临时存放的素材
✅ 不确定放哪里的东西

## 不要放收件箱

❌ 已经整理好的文件
❌ 立即需要处理的任务
❌ 个人隐私文件
❌ 系统配置文件
```

## 二、文件类型判断

### 2.1 类型识别

```markdown
## 文件类型判断流程

1. 看扩展名
   └─ .md / .txt / .html / .pdf / .docx / 图片...

2. 读文件内容
   └─ 文本文件 → 读取正文
   └─ 二进制文件 → 记录元信息

3. 判断主题
   └─ 从文件名 + 内容判断
   └─ 关联已有知识图谱
```

### 2.2 类型处理表

| 扩展名 | 处理方式 | 提取内容 | 归档位置 |
|--------|----------|----------|----------|
| .md | 直接读取 | 全文 | 按主题 |
| .txt | 直接读取 | 全文 | 按主题 |
| .html | 提取正文 | 去除标签 | 资料收集/ |
| .pdf | 文本提取 | 全文 | 按主题 |
| .docx | 文本提取 | 全文 | 按主题 |
| .xlsx | 数据提取 | 数据表 | 数据/ |
| .pptx | 文本提取 | 全文 | 资料/ |
| 图片 | OCR提取 | 文字内容 | 资料收集/ |
| 其他 | 记录元信息 | 文件名/大小/日期 | 待分类/ |

### 2.3 内容识别

```markdown
## 内容类型识别

📄 文档类
   ├─ 说明文档 → 知识沉淀/
   ├─ 教程指南 → 知识沉淀/
   └─ 报告总结 → 工作相关/

📊 数据类
   ├─ 数据报表 → 数据/
   ├─ 统计图表 → 数据/
   └─ 数据库 → 数据/

🖼️ 媒体类
   ├─ 截图/照片 → 资料收集/
   ├─ 音频/视频 → 资料收集/
   └─ 设计素材 → 资料收集/

💬 对话类
   ├─ 聊天记录 → 对话记录/
   └─ 邮件往来 → 对话记录/
```

## 三、分类标准

### 3.1 分类原则

```markdown
## 分类判断顺序

1. 看主题
   └─ 属于哪个领域？

2. 看用途
   └─ 用于工作？学习？个人？

3. 看类型
   └─ 文档？数据？素材？

4. 看关联
   └─ 和哪个项目相关？

5. 看频率
   └─ 常用？偶尔用？存档？
```

## 四、处理流程

### 4.1 标准流程

```
收到文件
    ↓
┌─────────────────────────────────────────┐
│  Step 1: 扫描                           │
│  ├─ 读取文件内容                        │
│  ├─ 识别文件类型                        │
│  └─ 提取元信息（名称/大小/日期）         │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  Step 2: 判断                           │
│  ├─ 判断文件类型（文档/数据/媒体）        │
│  ├─ 判断内容主题（查知识图谱）            │
│  └─ 判断紧急程度（常规/优先）             │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  Step 3: 分类                           │
│  ├─ 确定目标目录                        │
│  ├─ 检查是否重复                        │
│  └─ 确认归属正确                        │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  Step 4: 归档                           │
│  ├─ 移动文件到目标位置                  │
│  ├─ 重命名为规范格式                    │
│  └─ 创建索引记录（如需要）              │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  Step 5: 更新                           │
│  ├─ 更新知识图谱                        │
│  ├─ 更新MEMORY（如有必要）              │
│  └─ 记录处理日志                        │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│  Step 6: 报告                           │
│  ├─ 告知处理结果                        │
│  ├─ 说明归档位置                        │
│  └─ 提示关联知识                        │
└─────────────────────────────────────────┘
```

### 4.2 处理检查清单

```markdown

□ 文件已读取

□ 内容主题已判断
```

## 五、特殊情况处理

### 5.1 重复文件

```markdown
## 重复判断

- 文件名相同
- 文件内容相同
- 文件名不同但内容相同

## 处理方式

1. 保留最新版本
2. 将旧版本移到 历史/
3. 如不确定，询问用户
4. 记录处理日志
```

### 5.2 模糊分类

```markdown
## 模糊情况

- 无法判断主题
- 可能属于多个分类
- 内容不明确

## 处理方式

1. 先放"待分类/"
2. 标注待确认
3. 告知用户
4. 用户确认后重新分类
```

### 5.3 损坏文件

```markdown
## 损坏情况

- 文件无法读取
- 文件格式损坏
- 文件内容乱码

## 处理方式

1. 记录损坏信息
2. 尝试修复（如可能）
3. 移到"待处理/损坏文件/"
4. 告知用户
5. 等待用户指示
```

### 5.4 大文件

```markdown
## 大文件定义

- 单文件 > 100MB
- 或 > 50个文件

## 处理方式

1. 分批处理
2. 告知用户进度
3. 处理完一个批次再处理下一个
4. 记录处理状态
```
