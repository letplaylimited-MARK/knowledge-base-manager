# 工作流程 - 做事方式规范

> **这是AI的"操作手册"，定义了常见场景的处理方式**
> **格式：流程图 + 检查清单，便于AI执行**

---

## 一、入口分流

### 1.1 识别输入类型

```markdown
收到用户输入
    ↓
┌─────────────────────────────────────────┐
│                                         │
│  是新文件？    → [文件处理流程]          │
│  是任务指令？  → [任务执行流程]          │
│  是想法分享？  → [思绪整理流程]          │
│  是查询提问？  → [知识查询流程]          │
│  不确定？      → [询问确认]             │
│                                         │
└─────────────────────────────────────────┘
```

### 1.2 识别关键词

| 类型 | 关键词 | 触发流程 |
|------|--------|----------|
| 文件 | "文件"、"资料"、"上传"、"放进去" | 文件处理 |
| 任务 | "帮我"、"执行"、"做"、"完成" | 任务执行 |
| 想法 | "我想"、"有个想法"、"聊聊" | 思绪整理 |
| 查询 | "查一下"、"找找"、"告诉我" | 知识查询 |
| 反馈 | "不对"、"错了"、"不好用" | 问题反馈 |

### 1.3 不确定时的处理

```markdown
## 询问模板

🤔 我不确定你想做什么，可以再说明一下吗？

比如：
- "帮我整理这个文件" → 我知道是文件处理
- "我想聊聊AI的使用经验" → 我知道是思绪整理
- "帮我分析一下这个项目" → 我知道是任务执行
```

## 二、文件处理流程

### 2.1 流程图

```
收件箱
  ↓
┌─────────────────────────────────────────┐
│  1️⃣ 扫描 - 读取文件内容                 │
│     ├─ 判断文件类型                      │
│     ├─ 提取关键信息                      │
│     └─ 识别文件主题                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  2️⃣ 分类 - 归档到正确位置               │
│     ├─ 查询知识图谱                      │
│     ├─ 确定目标目录                      │
│     └─ 检查是否重复                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  3️⃣ 整理 - 更新相关文件                 │
│     ├─ 更新知识图谱                      │
│     ├─ 更新MEMORY                       │
│     └─ 更新项目文档（如相关）            │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  4️⃣ 报告 - 告知处理结果                 │
│     ├─ 文件已归档位置                    │
│     ├─ 识别的主题                        │
│     └─ 关联的知识                        │
└─────────────────────────────────────────┘
```

### 2.2 检查清单

```markdown
## 文件处理检查

□ 文件已读取
□ 文件类型已识别
□ 关键信息已提取
□ 目标目录已确定
□ 无重复文件
□ 文件已移动
□ 知识图谱已更新
□ MEMORY已更新
□ 用户已告知结果
```

### 2.3 文件类型处理

| 类型 | 处理方式 | 归档位置 |
|------|----------|----------|
| .md | 读取内容 | 按主题分类 |
| .txt | 读取内容 | 按主题分类 |
| .html | 提取正文 | 资料收集/ |
| .pdf | 提取文本 | 按主题分类 |
| 图片 | OCR提取 | 资料收集/ |
| .docx | 提取文本 | 按主题分类 |
| 其他 | 记录文件名 | 待分类/ |

## 三、任务执行流程

### 3.1 流程图

```
收到任务
  ↓
┌─────────────────────────────────────────┐
│  1️⃣ 理解 - 复述确认                     │
│     ├─ 复述任务目标                      │
│     ├─ 确认交付标准                      │
│     ├─ 确认时间要求                      │
│     └─ 识别资源限制                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  2️⃣ 计划 - 制定执行方案                 │
│     ├─ 拆解任务步骤                      │
│     ├─ 评估复杂度                        │
│     ├─ 选择角色协作                      │
│     └─ 预估时间资源                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  3️⃣ 执行 - 分步执行                     │
│     ├─ 按计划执行                        │
│     ├─ 定期汇报进度                      │
│     └─ 遇到问题及时反馈                  │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  4️⃣ 验证 - 检查结果                      │
│     ├─ 对照交付标准                      │
│     ├─ 检查完整性                        │
│     └─ 识别潜在问题                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  5️⃣ 交接 - 归档交付                     │
│     ├─ 产出物归档                        │
│     ├─ 任务记录归档                      │
│     ├─ 记忆同步更新                      │
│     └─ Git提交（如适用）                 │
└─────────────────────────────────────────┘
```

### 3.2 检查清单

```markdown
## 任务执行检查

### 理解阶段
□ 任务目标已复述
□ 交付标准已确认
□ 时间要求已确认
□ 资源限制已识别

### 计划阶段
□ 任务已拆解
□ 复杂度已评估
□ 角色已选定
□ 时间已预估

### 执行阶段
□ 按计划执行
□ 定期汇报
□ 问题已处理

### 验证阶段
□ 结果达标
□ 完整性检查
□ 问题已修复

### 交接阶段
□ 产出物归档
□ 任务记录归档
□ 记忆已更新
□ Git已提交
□ 用户已确认
```

### 3.3 汇报模板

```markdown
## 任务开始

✅ 收到任务：[任务简述]
📋 理解目标：[目标]
⏱️ 预计完成：[时间]

## 进度汇报

📊 进度：[已完成]/[总步骤]
⏱️ 预计剩余：[时间]

## 任务完成

✅ 完成！
📦 产出：[文件/结果]
📝 总结：[如有经验]

## 问题汇报

⚠️ 问题：[描述]
💡 建议：[处理方案]
```

## 五、知识查询流程

### 5.1 流程图

```
收到查询
  ↓
┌─────────────────────────────────────────┐
│  1️⃣ 理解 - 明确查询意图                 │
│     ├─ 复述查询问题                      │
│     ├─ 确认查询范围                      │
│     └─ 识别查询类型                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  2️⃣ 检索 - 搜索相关知识                 │
│     ├─ 查询MEMORY                       │
│     ├─ 查询知识图谱                      │
│     ├─ 检索知识沉淀                      │
│     └─ 检索对话历史                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  3️⃣ 整理 - 整合检索结果                 │
│     ├─ 筛选相关信息                      │
│     ├─ 排序优先级                        │
│     ├─ 标注来源                          │
│     └─ 识别知识缺口                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  4️⃣ 输出 - 给出答案                     │
│     ├─ 直接回答（如有答案）              │
│     ├─ 说明来源                          │
│     ├─ 标注不确定点                      │
│     └─ 询问是否需要更多信息              │
└─────────────────────────────────────────┘
```

### 5.2 查询类型处理

| 类型 | 处理方式 | 示例 |
|------|----------|------|
| 事实查询 | 直接回答 | "我的项目有哪些？" |
| 解释查询 | 整理回答 | "什么是七角色飞轮？" |
| 建议查询 | 分析建议 | "这个项目应该怎么做？" |
| 关联查询 | 整理关联 | "哪些知识和AI协作相关？" |

## 六、问题反馈流程

### 6.1 流程图

```
收到反馈
  ↓
┌─────────────────────────────────────────┐
│  1️⃣ 确认 - 复述问题                     │
│     ├─ 复述反馈内容                      │
│     ├─ 确认理解正确                      │
│     └─ 识别问题类型                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  2️⃣ 分析 - 诊断问题原因                 │
│     ├─ 是执行错误？                      │
│     ├─ 是理解错误？                      │
│     ├─ 是期望偏差？                      │
│     └─ 是系统限制？                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  3️⃣ 修复 - 提出解决方案                 │
│     ├─ 立即修复（如可能）                │
│     ├─ 制定修复计划                      │
│     └─ 征询用户意见                      │
└─────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────┐
│  4️⃣ 归档 - 记录经验                     │
│     ├─ 记录问题原因                      │
│     ├─ 记录解决方案                      │
│     ├─ 更新相关流程                      │
│     └─ 更新MEMORY                       │
└─────────────────────────────────────────┘
```

### 6.2 问题类型处理

| 类型 | 处理方式 | 负责人 |
|------|----------|--------|
| 执行错误 | 立即修复 | AI |
| 理解错误 | 重新理解 | AI+用户 |
| 期望偏差 | 沟通对齐 | AI+用户 |
| 系统限制 | 说明原因 | AI |

## 七、日常维护流程

### 7.1 每日维护

```markdown
## 每日开始

1. 检查收件箱
2. 回顾MEMORY
3. 确认今日任务
4. 开始工作

## 每日结束

1. 检查待办
2. 更新任务记录
3. 更新MEMORY
4. 记录经验（如有）
```

### 7.2 每周维护

```markdown
## 每周维护

1. 回顾本周产出
2. 整理知识沉淀
3. 更新知识图谱
4. 回顾MEMORY
5. 规划下周任务
```

## 八、紧急情况处理

### 8.1 紧急定义

```markdown
## 紧急情况

⚠️ 文件损坏/丢失
⚠️ 执行失败
⚠️ 数据不一致
⚠️ 用户急需结果
```

### 8.2 紧急处理

```markdown
## 处理原则

1. 立即告知用户
2. 评估影响范围
3. 优先恢复/修复
4. 事后总结经验

## 汇报模板

🚨 紧急情况：[描述]
📋 影响：[范围]
⏱️ 预计处理：[时间]
💡 方案：[方案]
```
