## 二、知识关联图谱

```
项目 ←→ 任务 ←→ 产出物
  ↑         ↓         ↓
  └─────────┴─────────┘
         记录

思绪 ←→ 讨论 ←→ 结晶
  ↑         ↓         ↓
  └─────────┴─────────┘
         归档

用户 ←→ AI ←→ 知识
  ↑         ↓         ↓
  └─────────┴─────────┘
         进化
```

### 2.2 知识点关联示例

```markdown
## 示例：AI协作方法

相关知识点：
- 思绪整理 → 使用

- 数据库管理 → 应用场景

- 最佳实践 → 来源
```

## 四、知识流转规则

### 4.1 流转方向

```
思绪（碎片）
    ↓ 整理
知识结晶（结构化）
    ↓ 验证
知识沉淀（稳定）
    ↓ 应用
经验总结（成熟）
    ↓ 归档
历史存档（参考）
```

### 4.2 流转触发

| 来源 | 触发条件 | 目标 | 动作 |
|------|----------|------|------|
| 对话 | 用户分享想法 | 思绪收集 | 记录 |
| 整理 | 思绪积累 | 知识结晶 | 结构化 |
| 执行 | 任务完成 | 经验库 | 归档 |
| 归档 | 知识过时 | 历史存档 | 标记 |

## 六、知识更新机制

### 6.1 更新触发

```markdown
## 何时更新

立即更新：
- 完成重要任务
- 用户明确偏好
- 重大决策

定期更新：
- 每日：任务记录
- 每周：知识整理
- 每月：系统回顾
```

### 6.2 更新规范

```markdown
## 更新格式

1. 添加内容：日期 + 内容 + 来源
2. 修改内容：日期 + 原内容 + 新内容 + 原因
3. 删除内容：日期 + 原内容 + 原因 + 移至

## 注意事项

- 保持结构清晰
- 标注更新时间
- 保留历史记录
```

## 七、知识沉淀流程

### 7.1 思绪收集

```markdown
触发：用户分享想法/经验/心得

流程：
1. 记录原始内容
2. 标注来源/时间
3. 关联已有知识
4. 存入思绪收集

位置：思绪收集-YYYYMMDD.md
```

### 7.2 知识结晶

```markdown
触发：思绪积累足够

流程：
1. 梳理核心要点
2. 结构化整理
3. 关联知识图谱
4. 评估成熟度
5. 归档到知识沉淀

位置：知识沉淀/[分类]/
```

### 7.3 知识落地

```markdown
触发：知识需要执行

流程：
1. 设计执行方案
2. 关联相关项目
3. 制定行动计划
4. 执行并记录
5. 更新经验库

产出：任务 + 产出物 + 经验
```
