# AI大模型配置说明

## 概述

本目录用于配置各种AI大模型的接入参数，使系统能够支持多种AI模型的统一调用。

## 配置文件结构

每个AI模型应有自己的配置文件，命名格式为：`{模型名称}.yaml`

## 支持的模型类型

1. OpenAI系列 (GPT-3.5, GPT-4, GPT-4 Turbo等)
2. Anthropic系列 (Claude 2, Claude 3, Claude 3 Opus等)
3. Google系列 (Gemini Pro, Gemini Ultra等)
4. Meta系列 (Llama 2, Llama 3等)
5. 深度求索系列 (DeepSeek-V2, DeepSeek-Coder等)
6. 智谱AI系列 (GLM-4, GLM-Turbo等)
7. 百川系列 (Baichuan2, Baichuan3等)
8. 零一万物系列 (Yi系列等)
9. 本地部署模型 (通过 Ollama, LMStudio, vLLM等)
10. 自定义API端点

## 配置文件格式

每个模型配置文件应包含以下字段：

```yaml
# 模型基本信息
model_name: "模型显示名称"
provider: "提供商名称"  # openai, anthropic, google, meta, deepseek, zhipu, baichuan, yi, ollama, custom
model_version: "具体模型版本"  # 如: gpt-4-turbo-preview

# API连接配置
api_key: "您的API密钥"  # 建议使用环境变量或安全 vault
api_base: "API基础URL"  # 如: https://api.openai.com/v1
api_version: "API版本"  # 如: v1 (可选)

# 模型参数
temperature: 0.7          # 采样温度 (0.0-2.0)
max_tokens: 2048          # 最大生成token数
top_p: 0.9               # nucleus sampling参数
frequency_penalty: 0.0   # 频率惩罚 (-2.0-2.0)
presence_penalty: 0.0    # 存在惩罚 (-2.0-2.0)

# 特殊功能支持
supports_streaming: true # 是否支持流式输出
supports_functions: true # 是否支持函数调用
supports_vision: false   # 是否支持视觉输入
supports_embedding: false # 是否支持嵌入向量生成

# 速率限制 (可选)
rate_limit:
  requests_per_minute: 60
  tokens_per_minute: 90000

# 代理设置 (可选)
proxy:
  http: "http://proxy.example.com:8080"
  https: "http://proxy.example.com:8080"
```

## 使用方法

1. 在本目录中创建对应的模型配置文件 (如: `openai-gpt4.yaml`)
2. 配置文件将被AI协作系统自动加载
3. 在智能体定义中引用模型名称来指定使用哪个模型
4. 系统将根据配置自动调用相应的API

## 环境变量支持

为提高安全性，建议在配置文件中使用环境变量引用敏感信息：

```yaml
api_key: "${OPENAI_API_KEY}"
```

系统将在加载配置时自动替换环境变量。

请参考本目录中的示例配置文件：
