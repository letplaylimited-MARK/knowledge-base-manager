# 任务验证标准

> 每次任务完成后必须执行的检查流程
> 版本：V1.0 | 日期：2026-04-11

---

## 一、验证检查清单

### 1.1 功能验证（必须）

```
□ 需求理解正确（复述用户需求）
□ 功能完整实现（对比需求清单）
□ 输出格式正确（符合约定）
□ 无明显错误（人工检查）
```

### 1.2 质量验证（必须）

```
□ 内容准确无误（事实核查）
□ 逻辑完整清晰（结构检查）
□ 语言表达通顺（阅读测试）
□ 无冗余/遗漏（完整性）
```

### 1.3 交接验证（必须）

```
□ 产出物已归档（指定目录）
□ 交接文档已生成（模板填充）
□ Git已提交（带清晰说明）
□ 记忆已更新（相关文件）
```

### 1.4 风险验证（必须）

```
□ 潜在风险已标注
□ 依赖项已说明
□ 已知限制已记录
□ 后续行动已明确
```

---

## 二、验证流程

```
任务执行完成
      ↓
启动验证流程
      ↓
┌─────────────────────────────────┐
│ 1. 功能验证                      │
│    → 需求对照检查                │
└─────────────────────────────────┘
      ↓ 通过？
┌─────────────────────────────────┐
│ 2. 质量验证                      │
│    → 内容和逻辑检查              │
└─────────────────────────────────┘
      ↓ 通过？
┌─────────────────────────────────┐
│ 3. 交接验证                      │
│    → 归档+文档+Git+记忆          │
└─────────────────────────────────┘
      ↓ 通过？
┌─────────────────────────────────┐
│ 4. 风险验证                      │
│    → 风险标注和说明              │
└─────────────────────────────────┘
      ↓ 完成
┌─────────────────────────────────┐
│ 5. 用户确认                      │
│    → 展示结果，等待确认          │
└─────────────────────────────────┘
```

## 四、验证触发条件

### 自动触发

```
✅ 任务执行完成
✅ 文件创建/修改
✅ 重要决策做出
✅ 每日工作结束
```

### 手动触发

```
✅ 用户要求验证
✅ 自己觉得需要检查
✅ 遇到不确定情况
```

## 五、与七角色的关系

| 角色 | 验证职责 |
|------|----------|
| 协调员 | 总体验证决策 |
| 评审员 | 主导质量验证 |
| 设计师 | 功能验收确认 |
| 观察员 | 记录改进建议 |

## 六、知识沉淀

每次验证后：

```
✓ 记录通过的验证点 → 经验库
✓ 记录发现的问题 → 问题案例库
✓ 记录改进建议 → 优化建议库
```
