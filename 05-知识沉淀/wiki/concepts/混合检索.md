# 混合检索

## 定义

混合检索（Hybrid Search）是指同时使用多种检索策略（通常为关键词检索+语义向量检索），并对两者的结果进行融合、去重和重排序，以获得比单一检索方式更优的召回率和精确率。

## 融合策略

### 1. 线性加权融合
```
final_score = α × keyword_score + (1-α) × semantic_score
```
其中 α 为关键词权重系数（默认 0.3）。

### 2. 倒数排名融合 (RRF)
```
RRF(d) = Σ 1/(k + rank_i(d))
```
- d: 文档
- rank_i: 文档在第 i 个检索器中的排名
- k: 平滑常数（通常 60）

### 3. 分层策略
先用向量检索缩小候选集，再在候选集上进行全文精排。

## 在本系统中的实现

本系统当前采用 **并列召回** 策略：

- **文件扫描层**: `search_content.py` 扫描文件名和文件内容
- **索引关键词层**: `vector_search.search_keyword()` 使用 SQLite `LIKE` 回退
- **语义层**: `vector_search.search()` 使用 FAISS `IndexFlatL2` + sentence-transformers
- **MCP 聚合层**: `mcp_server.handle_search_all()` 返回 `content_matches`、`filename_matches`、`vector_matches`

线性加权融合、RRF、BM25/FTS5 和 HNSW 属于可扩展设计方向，当前源码尚未实现。

## 检索流程

```
用户查询
    │
    ├──→ search_content 文件内容扫描 ──→ content_matches
    ├──→ search_content 文件名扫描 ──→ filename_matches
    └──→ vector_search FAISS/SQLite 回退 ──→ vector_matches
```

## 优势

1. **互补性**: 关键词检索擅长精确匹配，语义检索擅长模糊联想
2. **鲁棒性**: 单一检索器失效时仍有备选
3. **可调性**: 通过权重参数适应不同场景
4. **可扩展**: 可接入更多检索器（如图谱检索）

## 配置示例

```json
{
  "search": {
    "hybrid": true,
    "keyword_weight": 0.3,
    "semantic_threshold": 0.5,
    "top_k": 20,
    "dedup_method": "content_hash"
  }
}
```
