# 混合检索

## 定义

混合检索（Hybrid Search）是指同时使用多种检索策略（通常为关键词检索+语义向量检索），并对两者的结果进行融合、去重和重排序，以获得比单一检索方式更优的召回率和精确率。

## 融合策略

### 1. 线性加权融合
```
final_score = α × keyword_score + (1-α) × semantic_score
```
其中 α 为关键词权重系数（默认 0.3）。

### 2. 倒数排名融合 (RRF)
```
RRF(d) = Σ 1/(k + rank_i(d))
```
- d: 文档
- rank_i: 文档在第 i 个检索器中的排名
- k: 平滑常数（通常 60）

### 3. 分层策略
先用向量检索缩小候选集，再在候选集上进行全文精排。

## 在本系统中的实现

本系统采用 **线性加权融合** 策略：

- **关键词层**: SQLite FTS5 全文索引（BM25 相关性排序）
- **语义层**: FAISS HNSW 索引 + sentence-transformers 嵌入
- **融合层**: `.workbuddy/scripts/smart_router.py` 中的 `SmartRouter` 类
- **权重配置**: 通过 `config/` 目录的配置文件调整 α 值

## 检索流程

```
用户查询
    │
    ├──→ SQLite FTS5 关键词搜索 ──→ 结果集A（词项相关）
    │
    └──→ FAISS 向量语义搜索 ──→ 结果集B（语义相关）
                │
                ▼
          SmartRouter 融合
                │
                ▼
          去重 + 重排序
                │
                ▼
          最终结果集
```

## 优势

1. **互补性**: 关键词检索擅长精确匹配，语义检索擅长模糊联想
2. **鲁棒性**: 单一检索器失效时仍有备选
3. **可调性**: 通过权重参数适应不同场景
4. **可扩展**: 可接入更多检索器（如图谱检索）

## 配置示例

```json
{
  "search": {
    "hybrid": true,
    "keyword_weight": 0.3,
    "semantic_threshold": 0.5,
    "top_k": 20,
    "dedup_method": "content_hash"
  }
}
```
