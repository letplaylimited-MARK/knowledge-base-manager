# 记忆层级

## 定义

记忆层级（Memory Hierarchy）是模仿人类认知过程设计的三级记忆存储架构，包含短期记忆（STM）、中期记忆（MTM）和长期记忆（LTM），每一层有不同的存储策略、访问速度和持久化方式。

## 三级架构

### 短期记忆 (STM — Short-Term Memory)
- **存储方式**: 内存中的队列/缓存
- **容量**: 有限（默认最近 100 条交互）
- **生命周期**: 会话级别（进程重启后失效）
- **用途**: 当前会话上下文维护、话题追踪
- **与人的类比**: 工作记忆 — 你在对话中"正在想"的内容

### 中期记忆 (MTM — Mid-Term Memory)
- **存储方式**: SQLite 数据库（结构化表）
- **容量**: 中等（默认保留 30 天内容）
- **生命周期**: 跨会话持久化
- **用途**: 近期项目上下文、重复模式识别
- **检索方式**: 关键词 + 时间衰减加权
- **与人的类比**: 近期记忆 — "上周我们讨论过这个"

### 长期记忆 (LTM — Long-Term Memory)
- **存储方式**: FAISS HNSW 向量索引 + SQLite 元数据
- **容量**: 理论无限（受磁盘限制）
- **生命周期**: 永久持久化
- **用途**: 核心知识、项目经验、决策记录
- **检索方式**: 语义向量相似度搜索
- **与人的类比**: 远期记忆 — 经过"巩固"的知识

## 记忆流转

```
新信息 ──→ STM (临时缓存)
               │
               ▼ (相关性评分 > 阈值)
              MTM (中期存储)
               │
               ▼ (重要性评分 > 阈值 + 时间衰减后仍保留)
              LTM (长期向量索引)
               │
               ▼ (定期清理: 相关性 < 阈值)
             归档/遗忘
```

## 在本系统中的实现

- **核心引擎**: `.workbuddy/记忆层/memoryos.py` 中的 `MemoryOS` 类
- **存储路径**: `.workbuddy/memory_data/`
- **索引格式**: FAISS 二进制索引文件 (`.faiss` / `.index`)
- **管理命令**: `run_maintenance` (MCP工具) 触发记忆维护

## 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| stm_max_items | 100 | 短期记忆最大条目 |
| mtm_retention_days | 30 | 中期记忆保留天数 |
| ltm_consolidation_threshold | 0.7 | 升级为长期记忆的相关性阈值 |
| maintenance_interval_hours | 24 | 记忆维护间隔 |
