# 记忆层级

## 定义

记忆层级（Memory Hierarchy）是模仿人类认知过程设计的三级记忆存储架构，包含短期记忆（STM）、中期记忆（MTM）和长期记忆（LTM），每一层有不同的存储策略、访问速度和持久化方式。

## 三级架构

### 短期记忆 (STM — Short-Term Memory)
- **存储方式**: 内存中的队列/缓存
- **容量**: 默认 7 条
- **生命周期**: 当日 JSONL 持久化 + 进程内队列
- **用途**: 当前会话上下文维护、话题追踪
- **与人的类比**: 工作记忆 — 你在对话中"正在想"的内容

### 中期记忆 (MTM — Mid-Term Memory)
- **存储方式**: 按记忆类型分组的 JSON 文件
- **容量**: 默认 1000 条
- **生命周期**: 跨会话持久化
- **用途**: 近期项目上下文、重复模式识别
- **检索方式**: 关键词 + 时间衰减加权
- **与人的类比**: 近期记忆 — "上周我们讨论过这个"

### 长期记忆 (LTM — Long-Term Memory)
- **存储方式**: `user_profile`、`knowledge`、`strategies` 目录下的 JSON 文件
- **容量**: 默认知识 100 条，策略 50 条
- **生命周期**: 永久持久化
- **用途**: 核心知识、项目经验、决策记录
- **检索方式**: 当前实现为关键词包含匹配 + 上下文汇总
- **与人的类比**: 远期记忆 — 经过"巩固"的知识

## 记忆流转

```
新信息 ──→ STM (临时缓存)
               │
               ▼ (相关性评分 > 阈值)
              MTM (中期存储)
               │
               ▼ (重要性评分 > 阈值 + 时间衰减后仍保留)
              LTM (长期向量索引)
               │
               ▼ (定期清理: 相关性 < 阈值)
             归档/遗忘
```

## 在本系统中的实现

- **核心引擎**: `.workbuddy/记忆层/memoryos.py` 中的 `MemoryOS` 类
- **存储路径**: `.workbuddy/记忆层/memory_data/`
- **索引格式**: JSONL/JSON 文件；未使用 FAISS/HNSW 作为 MemoryOS 存储
- **管理命令**: `run_maintenance` (MCP工具) 触发记忆维护

## 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| short_term_capacity | 7 | 短期记忆最大条目 |
| mid_term_capacity | 1000 | 中期记忆最大条目 |
| long_term_knowledge_capacity | 100 | 长期知识最大条目 |
| mid_term_heat_threshold | 5.0 | 晋升长期记忆的热度阈值 |
